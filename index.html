AI Context & Memory, Step by Step

A pedagogical walkthrough of how modern AI systems manage context windows, memory, compaction, retrieval, and caching.

1) Context vs memory

Context is what the model sees right now in one request. Memory is what the system stores and can retrieve later.

2) Why forgetting happens

Context windows are finite. Long sessions require pruning, summarization, retrieval, or reset.

3) Compaction

Compaction summarizes older conversation and keeps recent turns detailed. This preserves signal while reducing token load.

4) Retrieval

Good retrieval injects small, relevant snippets. Bad retrieval floods context with noisy text.

5) Prompt caching

Stable prompt prefixes can be reused to reduce latency and token cost.

6) Platform view

  • ChatGPT: context + optional memory features
  • Claude/Claude Code: file memory + prompt caching
  • Codex: project/user config, agent workflows
  • OpenClaw: JSONL sessions + compaction + memory tools
